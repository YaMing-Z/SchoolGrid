# SchoolGrid 整体向导式布局实施计划 (Stepper Implementation Plan)

本文档整理了将当前系统优化为"向导式（Stepper / Wizard）"流程的具体重构与实施步骤。旨在为用户提供清晰、线性的全校排课体验。

> 最后更新：2026-03-21

|---|---|---|

## 1. 整体架构与状态机调整

目前的系统主要通过 `currentView` 进行页面切换。为了支持向导式操作，需要在全局 Store 中进行流转状态管理。

- **新增状态（`src/stores/scheduleStore.ts`）**:
  - `currentStep: number`，表示当前流程阶段，初始值为 `0`。
  - 各步骤定义：
    - **Step 0**: 导入基础数据 (`ImportPage`)
    - **Step 1**: 配置排课规则 (`RuleConfigPage`)
    - **Step 2**: 自动生成课表 (`ScheduleGenerationPage`，若暂无独立页面也可在后台流转并跳过)
    - **Step 3**: 课表微调与导出 (`ScheduleGrid` & `AdjustmentPanel`)

- **修改 `App.tsx` 页面结构**:
  - 在内容区顶部引入横向 `Stepper`（步骤条）组件，直观显示当前的 `[导入数据 -> 规则配置 -> 自动排课 -> 微调导出]` 进度。
  - 在页面底部或各页面内容底部统一增加导航按钮（"上一步" / "下一步"）。

## 2. 各步骤具体实施点与重构拆解

### Step 0: 导入基础数据阶段

- **现状**：代码已经具备 Excel 导入读取，且提供了模板下载。
- **待实现**：
  - [ ] 数据导入完成后，展示概览与校验信息（如："识别到 XX 位教师、XX 条教学任务"）
  - [ ] 在无致命数据冲突时，允许用户点击"下一步"进入规则配置阶段

### Step 1: 规则配置阶段

- **现状**：已有 `RuleConfigPage`，支持增删改查规则。规则引擎已全面集成（见 [rule_engine_audit.md](./rule_engine_audit.md)）。
- **待实现**：
  - [ ] **动态生成默认规则**：导入数据后，系统自动为"语文"、"数学"、"英语"等关键科目生成默认基础规则（保持单科目级别的"无连堂"设置）
  - [ ] **教师特殊时间约束网格**：新增交互式二维网格，行为教师、列为时段，点击格子设置"不可排课"等约束
  - [ ] 配置确认后，通过"开始排课 / 下一步"进入算法测算

### Step 2: 自动排课阶段

- **待实现**：
  - [ ] 点击开始排课时，界面切换至加载状态，显示进度圆环或动效
  - [ ] **软降级（容错）处理机制**：不再因部分硬冲突直接报排课失败，将无法排入的课时归入"未排课程池（暂存区）"
  - [ ] 排课结束后自动跳转至 Step 3，若不完整则横幅提示"遗留 X 节冲突课程待手动微调"

### Step 3: 微调与导出阶段

- **现状**：已有 `ScheduleGrid` 和 `AdjustmentPanel`，调课引擎已集成学科禁排规则检查。
- **待实现**：
  - [ ] **冲突高亮联动校验**：拖拽调整时，前端实时预校验目标格子状态；发生教师重叠或违反禁排规则时，格子闪红阻止放置
  - [ ] **导出闭环**：一键导出 Excel，固化最终课表

## 3. 未完成开发任务清单

> 按优先级从高到低排列

### 🔴 高优先级（核心流程）

- [ ] **[向导式布局]** 新增 `Stepper` 组件，重写 `App.tsx` 页面框架，全局 Store 加入 `currentStep` 步骤流转状态
- [ ] **[软降级排课]** 在 `greedyScheduler` 中引入"未排课程池"暂存区，实现"尽力而为"排课，不再直接阻断
- [ ] **[拖拽冲突校验]** 在 `ScheduleGrid` 拖拽交互中补齐实时冲突高亮（教师重叠 / 学科禁排 / 固定课位）
- [ ] **[导出功能]** 一键导出 Excel，完成全流程闭环

### 🟡 中优先级（体验增强）

- [ ] **[默认规则补全]** 导入数据后自动为主科生成基础排课规则
- [ ] **[导入概览]** 数据导入成功后展示"识别到 XX 位教师、XX 条任务"等校验摘要
- [ ] **[教师约束网格]** 交互式时间约束网格组件（行：教师，列：时段，点击设置约束）
- [ ] **[TeacherTimeRule prefer_not]** 排课算法中为教师"尽量不排"时段加入软惩罚分数（评分降权，不硬拦截）

### 🟢 低优先级（锦上添花）

- [ ] **[调课历史]** 调课历史记录与撤销功能
- [ ] **[批量调课]** 一次处理多节课的批量调课支持
- [ ] **[多套方案]** 多套课表方案对比，课表版本管理
- [ ] **[打印优化]** A4 纸张适配打印样式
- [ ] **[性能]** 大数据量（100+ 班级）性能优化，算法执行进度显示

# SchoolGrid：智能排课系统需求计划书

## 1. 项目概述

构建一个轻量化 Web 排课工具，核心解决“学期基准排课”与“日常临时调课”两大场景。

- **架构**：Master（基准表）+ Delta（变动记录）。
- **特性**：支持复杂教师 - 班级配比、自定义规则引擎、三级优先级调课算法。

---

## 2. 数据导入与模板设计 (Data Ingestion & Templates)

系统需提供标准的 Excel 模板下载，建议采用 **多工作表 (Multi-Sheet)** 或 **关联导入** 方案。

### 2.1 模板结构

为了解决“数学老师带 2 个班，体育老师带 6 个班”的问题，将数据拆解为三个关联维度：

1. **教师表 (Teachers)**：
    - 字段：工号、姓名、所属学科、周总课时上限、避开时段。
    - _说明：这里只记录老师的个人属性。_
2. **班级表 (Classes)**：
    - 字段：年级、班级编号、班主任。
3. **教学计划表 (Curriculum Plan - 核心映射)**：
    - 字段：班级编号、学科、**授课教师姓名/工号**、周课时数、是否连堂。
    - **逻辑实现**：
        - 若“张老师”教“初二 1 班”和“初二 2 班”的数学，则在表里出现两行记录。
        - 体育老师同理，会出现 6 行对应不同班级的记录。
        - **Agent 逻辑**：系统在解析此表时，会自动计算每位老师的总工作量，并与教师表的“课时上限”做校验。

### 2.2 用户交互

- **模板下载**：在“数据初始化”页面提供 `SmartSchedule_Template.xlsx` 下载。
- **导入预检**：上传后，AI Agent 需检查：
  - 是否存在“老师总课时超过上限”的情况。
  - 是否存在“班级总课时超过课表格子总数”的情况。

---

## 3. 规则引擎 (Rule Engine - Redmine Style)

采用可插拔的过滤器设计，用户可自由组合逻辑：

- **规则维度**：
  - **时间约束**：如“XX 课必须在上午/下午”、“XX 老师周五不可排课”。
  - **频率约束**：如“数学课每天最多 1 节”、“体育课每周必须有一次连堂”。
  - **并发约束**：如“同一时段实验室只能容纳 2 个班级”。
- **效力分级**：
  - **硬约束 (Strict)**：违反则排课失败。
  - **软约束 (Flexible)**：允许存在，但系统会通过 UI 标记警告。

---

## 4. 智能调课与缺勤逻辑 (Daily Adjustment)

当发生临时调课（如老师请假）时，系统执行以下 **Master-Delta** 逻辑：

### 4.1 自动寻优优先级

1. **P0：同班同日互换**（对学生影响最小，不影响家长接送和学生作业进度）。
2. **P1：同班本周内跨日互换**（保证班级内部课时总量不变）。
3. **P2：跨老师同科目代课**（最后的保底手段）。

### 4.2 调课辅助功能

- **建议列表**：系统自动列出 P0-P2 的所有可行方案，并标注“符合所有规则”或“违反 [XX] 规则”。
- **受影响视图**：选中一个建议后，实时预览该班级和相关老师的课表 Diff（变动前后对比）。

---

## 5. 技术栈与分享机制 (Tech Stack)

- **前端**：React + Tailwind CSS + Vite。
- **状态管理**：Zustand（存储 Master 课表和 Delta 数组）。
- **Excel 处理**：`SheetJS (js-xlsx)`（负责模板读取与结果导出）。
- **持久化**：`LocalStorage` 存储草稿，支持导出 `.schedule` (JSON 格式) 文件用于分享或存档。
